# Hoshi Messaging Design

## Purpose

Define the v1 messaging system for Hoshi so landing pages, docs, decks, demos, and developer materials all use the same language.

## One-Liner

Hoshi helps AI agents pay safely and get paid on Solana.

## Tagline

Safe payments for AI agents.

## Hero Options

### Option 1

Safe payments for AI agents.

Subtext:
- Hoshi helps AI agents pay safely and get paid on Solana.

### Option 2

Give your AI agent a safe way to pay and get paid.

Subtext:
- Built on Solana with support for core agent payment standards.

### Option 3

Payment infrastructure for AI agents on Solana.

Subtext:
- Simple to integrate, protocol-aligned, and designed for safe execution.

## Subheadline Options

- Add safe payment rails to your AI agent with x402 and MPP support on Solana.
- Hoshi gives agent apps and developers a focused payment layer instead of a generic crypto copilot.
- Built for teams that need agent-native payments, not broad DeFi complexity.
- Start with safe agent payments today and grow into richer agent commerce later.

## Problem Statement

AI agents can take actions, call tools, and complete workflows, but most payment systems are still built for humans. That creates friction for agent developers and risk for products trying to let agents transact. Teams need a focused payment layer that is safe, protocol-aware, and easy to integrate.

## Solution Statement

Hoshi gives AI agents a safe way to pay and get paid on Solana. It provides focused payment infrastructure with support for x402 and MPP, so agent builders can add payment capabilities without turning their product into a custom crypto stack.

## What Hoshi Is

- Agent payment infrastructure on Solana.
- A safe execution layer for agent payments.
- A developer product for consumer AI agent apps and agent builders.
- A simple product story backed by protocol-aligned infrastructure.
- Solana-native at the core, with APP optional later rather than required for the base story.

## What Hoshi Is Not

- Not a generic DeFi copilot.
- Not a trading assistant.
- Not a portfolio manager.
- Not an agent shell or agent-first clone of another product.
- Not a broad autonomous commerce platform in v1.

## Feature Bullets

- Send payments from AI agents with safer execution controls.
- Let agents get paid for actions, services, or completed work.
- Support x402 and MPP as core payment standards.
- Integrate through a developer-friendly payment layer on Solana.
- Start narrow with payments, with room for future workflow expansion.

## Developer-Facing Copy

### Short version

Hoshi is payment infrastructure for AI agents on Solana. Use it to add safe outbound and inbound payment flows with x402 and MPP support.

### Docs intro version

Hoshi helps developers give AI agents a safe way to pay and get paid on Solana. Instead of assembling payment logic from scratch, teams can integrate a focused infrastructure layer built for agent-native payment flows.

### Integration framing

- Add payment capabilities without building a full crypto payments stack.
- Use protocol-aligned primitives instead of ad hoc payment wiring.
- Keep the product surface simple while relying on stronger infrastructure underneath.

## Messaging Guardrails

- Lead with payments, not abstract agent commerce.
- Always keep "safe" close to the primary value proposition.
- Mention Solana explicitly in core copy, especially in product and developer contexts.
- Use x402 and MPP as credibility anchors, not jargon overload.
- Treat APP as optional future expansion, not required v1 positioning.
- Avoid language that implies Hoshi is a generic assistant, copilot, or trading product.
- Avoid language that makes Hoshi sound like a clone of another agent finance product.
- Avoid leading with roadmap ideas like negotiation, shopping, or escrow.
- Keep top-line copy simple enough for non-technical buyers.
- Reserve deeper technical detail for docs, architecture, and developer materials.

## Internal Messaging Boundary

- External copy stays simple: safe payments for AI agents on Solana.
- Internal product, roadmap, and developer narratives can explicitly reference the long-term `sdk`, `engine`, and `mcp` depth target.
- When deeper architecture is mentioned, frame it as Solana-native evolution around x402 and MPP, not as a brand or identity copy of another product.

## Preferred Messaging Shape

Use this order in most external assets:
1. Safe payments for AI agents.
2. Hoshi helps AI agents pay safely and get paid on Solana.
3. x402 and MPP support for agent developers and consumer agent apps.
4. Future workflow expansion exists, but is not the v1 headline.
